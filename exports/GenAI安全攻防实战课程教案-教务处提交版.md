## 二、教案总体设计

### 1. 学情分析

学生具备 Python 编程和基础网络安全知识，但对生成式 AI 的系统风险认知和工程化攻防能力普遍不足。课程通过“攻击复现 + 防御构建 + 综合审计”路径，提升学生从概念理解到实战落地的能力。

### 2. 教学目标

1. 知识目标：掌握提示词攻击、防御工程、对抗样本、隐私泄露、数据投毒、供应链安全和安全评估方法。  
2. 能力目标：能够在云环境中独立完成 AI 应用安全测试、防护验证和报告输出。  
3. 素质目标：形成合法授权测试意识、负责任披露意识和证据化工作习惯。  

### 3. 教学重点与难点

1. 教学重点：提示词攻击链路、输入/输出防护、多层防御整合、STRIDE 威胁建模与风险矩阵。  
2. 教学难点：攻击技术与防御策略的映射关系、综合审计中的证据完整性、误报与漏报平衡。  

### 4. 教学方法与手段

1. 讲授法：建立系统概念框架。  
2. 案例法：通过真实事件分析风险成因。  
3. 任务驱动法：以实验任务推进能力形成。  
4. 演示 + 实操法：教师演示关键步骤，学生独立复现并提交证据。  

### 5. 课程思政与职业素养融入

1. 在每次攻防实验中强调“授权边界、合法测试、责任披露”。  
2. 在伦理与合规模块强化数据安全、社会责任与职业操守。  
3. 在综合审计阶段强调团队协作沟通与工程文档规范。

2. 攻击复现证据：20分。  
3. 防御设计与验证：20分。  
4. 风险评级与整改建议：10分。  
5. 合规与伦理说明：10分。  
6. 可复现性与文档规范：25分。
